# oxpulse-sfu-kit v0.3.0 — str0m Encapsulation Pass

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Hide str0m types from the public API so that str0m minor bumps (0.18 → 0.19 → …) no longer require a breaking release of oxpulse-sfu-kit. Internal modules keep using str0m directly — the pass is strictly at the `pub fn` / `pub enum` boundary.

**Architecture:** Boundary-wrap pattern. Each leaky public site gets a newtype or wrapper struct with `From<&str0m::X>` (inbound) and `Into<str0m::X>` (outbound) conversions. Internal impl code stays unchanged. A small `pub mod raw { ... }` escape hatch re-exports `str0m::Rtc` and is explicitly marked as semver-exempt.

**Tech Stack:** Rust 1.86 MSRV, str0m 0.18, existing deps. Zero new crate dependencies.

**Reference context:**
- Audit of current leakage: see parent summary in docs/superpowers/plans/audit notes (Rtc, Input, Transmit, MediaData, KeyframeRequest, Rid, Mid, Pt).
- Motivation: str0m discussion #944 (both Thomas Eizinger and Martin Algesten echoed the semver-encapsulation argument).
- Security posture: `#![forbid(unsafe_code)]` already crate-wide. New modules MUST inherit this. Add `#[non_exhaustive]` to every new public enum to reserve semver headroom. No `pub` fields on wrapper structs — force accessor methods so internal representation can change without breaking.

**Out of scope for v0.3.0:**
- test-utils feature will stay str0m-leaky (documented as semver-exempt).
- BandwidthEstimator/Pacer port (deferred to v0.4).
- AV1 Dependency Descriptor / SFrame — deferred.

//! Opaque newtype wrappers for str0m identifier types.
//!
//! These exist to prevent str0m semver churn from propagating to downstream
//! consumers. Internal modules keep using str0m types directly.

use std::fmt;
use std::str::FromStr;

/// Simulcast layer identifier (RFC 8852 RID).
///
/// Constructors accept the conventional `"q"` / `"h"` / `"f"` quality tags
/// plus any other short ASCII string str0m accepts.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub struct SfuRid(str0m::media::Rid);

/// Media stream identifier within a single peer connection.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub struct SfuMid(str0m::media::Mid);

/// RTP payload type (codec binding within a session).
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub struct SfuPt(str0m::media::Pt);

impl SfuRid {
    /// Construct from a `str` slice (RID is a short ASCII identifier).
    pub fn from_str(s: &str) -> Result<Self, InvalidRid> {
        str0m::media::Rid::from_str(s)
            .map(SfuRid)
            .map_err(|_| InvalidRid)
    }
}

impl fmt::Display for SfuRid {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        fmt::Display::fmt(&self.0, f)
    }
}

/// Error returned when a string cannot be parsed as a [`SfuRid`].
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct InvalidRid;

impl fmt::Display for InvalidRid {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        f.write_str("invalid RID: must be short ASCII")
    }
}

impl std::error::Error for InvalidRid {}

// Internal-only bidirectional conversion. `pub(crate)` so internal modules can
// cross the boundary without exposing str0m types to downstream.
impl SfuRid {
    pub(crate) fn from_str0m(r: str0m::media::Rid) -> Self {
        Self(r)
    }
    pub(crate) fn to_str0m(self) -> str0m::media::Rid {
        self.0
    }
}

impl SfuMid {
    pub(crate) fn from_str0m(m: str0m::media::Mid) -> Self {
        Self(m)
    }
    pub(crate) fn to_str0m(self) -> str0m::media::Mid {
        self.0
    }
}

impl SfuPt {
    pub(crate) fn from_str0m(p: str0m::media::Pt) -> Self {
        Self(p)
    }
    pub(crate) fn to_str0m(self) -> str0m::media::Pt {
        self.0
    }
}

///

    pt: SfuPt,

//! UDP datagram wrappers.

use std::net::SocketAddr;
use std::time::Instant;

/// Transport protocol for an SFU datagram.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
#[non_exhaustive]
pub enum SfuProtocol {
    /// Plain UDP.
    Udp,
    /// UDP over TCP (ICE-TCP).
    Tcp,
    /// SSL over TCP (TURN).
    SslTcp,
    /// TLS over TCP (TURN).
    Tls,
}

impl SfuProtocol {
    pub(crate) fn from_str0m(p: str0m::net::Protocol) -> Self {
        use str0m::net::Protocol;
        match p {
            Protocol::Udp => Self::Udp,
            Protocol::Tcp => Self::Tcp,
            Protocol::SslTcp => Self::SslTcp,
            Protocol::Tls => Self::Tls,
        }
    }
    pub(crate) fn to_str0m(self) -> str0m::net::Protocol {
        use str0m::net::Protocol;
        match self {
            Self::Udp => Protocol::Udp,
            Self::Tcp => Protocol::Tcp,
            Self::SslTcp => Protocol::SslTcp,
            Self::Tls => Protocol::Tls,
        }
    }
}

/// A datagram received from the network, ready to feed into an `SfuRtc`.
#[derive(Debug)]
pub struct IncomingDatagram {
    /// Wall-clock time the datagram was received.
    pub received_at: Instant,
    /// Transport protocol.
    pub proto: SfuProtocol,
    /// Remote address.
    pub source: SocketAddr,
    /// Local bound address.
    pub destination: SocketAddr,
    /// Raw wire bytes.
    pub contents: Vec<u8>,
}

/// A datagram the SFU wants to send.
#[derive(Debug)]
pub struct OutgoingDatagram {
    /// Transport protocol.
    pub proto: SfuProtocol,
    /// Source (local) address.
    pub source: SocketAddr,
    /// Destination (remote) address.
    pub destination: SocketAddr,
    /// Wire bytes.
    pub contents: Vec<u8>,
}

impl OutgoingDatagram {
    pub(crate) fn from_transmit(t: str0m::net::Transmit) -> Self {
        Self {
            proto: SfuProtocol::from_str0m(t.proto),
            source: t.source,
            destination: t.destination,
            contents: t.contents,
        }
    }
}
```

//! Opaque SFU-owned handle over a str0m `Rtc` state machine.

/// An opaque WebRTC peer connection state machine.
///
/// Construct via [`SfuRtcBuilder`]. Pass to [`Client::new`][crate::Client::new].
///
/// The underlying str0m type is deliberately hidden. If you need functionality
/// not exposed by the builder, see [`crate::raw`] (semver-exempt).
pub struct SfuRtc(pub(crate) str0m::Rtc);

impl std::fmt::Debug for SfuRtc {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        f.debug_struct("SfuRtc").finish_non_exhaustive()
    }
}

/// Builder for [`SfuRtc`] exposing the common configuration subset.
///
/// For advanced configuration (custom codec formats, non-standard extensions,
/// custom schedulers), drop down to [`crate::raw::rtc_builder`] — but note
/// that escape hatch is **not** covered by oxpulse-sfu-kit's semver guarantee.
#[derive(Debug, Default)]
pub struct SfuRtcBuilder {
    // We keep a raw str0m builder internally. This module is the only place
    // that directly references it, so any str0m API churn lands here and
    // nowhere else in downstream code.
    inner: str0m::RtcConfig,
}

impl SfuRtcBuilder {
    /// Start a new builder with str0m defaults.
    pub fn new() -> Self {
        Self { inner: str0m::RtcConfig::default() }
    }

    /// Enable receiving statistics events.
    pub fn enable_stats(mut self, enabled: bool) -> Self {
        self.inner = self.inner.set_stats_interval(
            if enabled { Some(std::time::Duration::from_secs(1)) } else { None }
        );
        self
    }

    /// Enable transport-wide congestion control (TWCC) and egress BWE.
    pub fn enable_bwe(mut self, initial_bitrate_bps: u64) -> Self {
        self.inner = self.inner.enable_bwe(Some(str0m::Bitrate::from(initial_bitrate_bps)));
        self
    }

    /// Finish the builder.
    pub fn build(self) -> SfuRtc {
        SfuRtc(self.inner.build())
    }
}
```

## Task 8: Encapsulation surface test

**Files:**
- Create: `tests/encapsulation_surface.rs`

**Why a test and not just clippy:** there is no stable lint that catches "public item mentions foreign crate type". We grep the generated rustdoc JSON instead — any `str0m::` in public docs = regression.

- [ ] **Step 1: Write the test**

Create `tests/encapsulation_surface.rs`:
```rust
//! Compile-time surface test: no str0m types in public docs.
//!
//! Runs `cargo rustdoc` with JSON output and asserts the public API does not
//! reference `str0m::` anywhere outside of the `raw::` escape-hatch module.

#![cfg(not(target_env = "msvc"))] // rustdoc json is flaky on MSVC in CI

use std::process::Command;

#[test]
fn public_api_does_not_leak_str0m_types() {
    let out = Command::new("cargo")
        .args(["+nightly", "rustdoc", "--lib", "--", "-Zunstable-options", "--output-format=json"])
        .output()
        .expect("run cargo rustdoc");
    // If nightly isn't available, skip rather than fail.
    if !out.status.success() {
        eprintln!("skipping: cargo +nightly rustdoc failed (nightly toolchain absent?)");
        return;
    }
    let path = std::path::Path::new("target/doc/oxpulse_sfu_kit.json");
    if !path.exists() {
        eprintln!("skipping: rustdoc JSON not produced");
        return;
    }
    let content = std::fs::read_to_string(path).expect("read rustdoc json");
    // Strip the `raw::` module section before grep: count of "str0m::" occurrences
    // outside that scope must be zero. Simplest approach: fail if we find str0m::
    // in any path whose segments[0] != "raw".
    //
    // Exact JSON parsing is noisy; the contract we want to hold is that no
    // public doc link points at a str0m item from a non-raw path. A substring
    // count is a crude but effective proxy for now.
    let non_raw_leaks: usize = content
        .split("\"path\":")
        .filter(|segment| segment.contains("str0m::"))
        .filter(|segment| !segment.contains("\"raw\""))
        .count();
    assert_eq!(non_raw_leaks, 0, "str0m types leak through non-raw public paths");
}
```

Run: `cargo test --test encapsulation_surface`
Expected: PASS (or skipped if nightly absent). If PASS — good. If FAIL, grep the JSON to find which public item still mentions str0m and wrap it.

## [0.3.0] - 2026-04-23

### Breaking

- **str0m encapsulation pass.** Public API no longer exposes `str0m::*` types directly.
  - `Propagated::MediaData(ClientId, MediaData)` → `Propagated::MediaData(ClientId, SfuMediaPayload)`
  - `Propagated::KeyframeRequest(ClientId, KeyframeRequest, ClientId, Mid)` → uses `SfuKeyframeRequest` + `SfuMid`
  - `Client::new(str0m::Rtc, ...)` → `Client::new(SfuRtc, ...)`
  - `Client::handle_input(str0m::Input)` → `Client::handle_input(IncomingDatagram)`
  - `Client::accepts(&str0m::Input)` → `Client::accepts(&IncomingDatagram)`
  - `Client::drain_pending_out()` now yields `OutgoingDatagram` instead of re-exported `str0m::net::Transmit`
  - `Client::desired_layer()` / `set_desired_layer()` / `active_rids()` now use `SfuRid`
  - `pub type Transmit = str0m::net::Transmit` removed
- **Escape hatch**: use `oxpulse_sfu_kit::raw::*` if you need access to raw str0m types. The `raw` module is explicitly semver-exempt — minor str0m bumps may change it without a major bump here.

### Added

- `SfuRid`, `SfuMid`, `SfuPt` — newtype wrappers for str0m identifier types.
- `SfuMediaPayload`, `SfuMediaKind` — media payload + kind wrappers.
- `SfuKeyframeRequest`, `SfuKeyframeKind` — keyframe-request wrappers.
- `IncomingDatagram`, `OutgoingDatagram`, `SfuProtocol` — datagram wrappers.
- `SfuRtc`, `SfuRtcBuilder` — opaque Rtc handle + façade builder exposing ICE servers, BWE, stats.
- `raw` module — semver-exempt escape hatch for advanced users.
- `tests/encapsulation_surface.rs` — compile-time check that no non-`raw::` public item references str0m types.

### Migration

Most downstream code changes are mechanical:

```rust
// before (v0.2)

let rtc = Rtc::new();
let client = Client::new(rtc, metrics);

// after (v0.3)

```

```rust
// before

    Propagated::MediaData(id, data) => forward(data.mid, &data.data),
    ...
}

// after
match propagated {
    Propagated::MediaData(id, payload) => forward(payload.mid(), payload.data()),
    ...
}
```

### Rationale

Per [str0m#944](https://github.com/algesten/str0m/discussions/944): str0m pre-1.0 minor bumps are breaking. Pinning our public API to their types propagated that churn to downstream consumers. Hiding str0m behind wrappers keeps our semver independent.
```
